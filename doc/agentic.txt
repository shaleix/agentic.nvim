INTRODUCTION                                      *agentic-introduction*

                                                           *agentic.nvim*

Agentic.nvim brings your AI assistant to Neovim through the Agent Client
Protocol (ACP). You get the same results and performance as the provider's
official CLI directly from the terminal.

Agentic.nvim is the interface; your agent is the brain. The plugin uses
your existing terminal configurations and authentication methods --
including MCP servers, commands, skills, and sub-agents.

Sessions are interchangeable: start a conversation in Neovim and continue
it in the terminal, or pick up a terminal session inside Neovim. Your ACP
provider manages sessions natively, so they are available everywhere.

There are no hidden prompts or magic behind the scenes -- just a chat
interface, your colors, and your keymaps.

Supported providers:
  - Claude (claude-agent-acp)
  - Gemini (gemini-acp)
  - Codex (codex-acp)
  - OpenCode (opencode-acp)
  - Cursor Agent (cursor-acp)
  - Copilot (copilot-acp)
  - Augment (auggie-acp)
  - Mistral Vibe (mistral-vibe-acp)
  - Cline (cline-acp)
  - Goose (goose-acp)
  - ...and any future ACP-compatible provider

REQUIREMENTS                                      *agentic-requirements*

- Neovim v0.11.0 or higher
- An ACP provider CLI (must be in PATH or an absolute path)

For security reasons, this plugin does not install or manage binaries.
You must install the CLI for your chosen provider manually. Global
install is recommended (especially with `pnpm`, which uses a stable
global path), but any method that puts the binary in PATH works.

Pin your versions! See the security note below each provider.

INSTALLATION                                      *agentic-installation*

                                                       *agentic.setup()*

Using lazy.nvim: ~
>lua
  {
    "carlos-algms/agentic.nvim",

    --- @type agentic.PartialUserConfig
    opts = {
      -- Any ACP-compatible provider works. Built-in:
      -- "claude-agent-acp" | "gemini-acp" | "codex-acp"
      -- "opencode-acp" | "cursor-acp" | "copilot-acp"
      -- "auggie-acp" | "mistral-vibe-acp" | "cline-acp"
      -- "goose-acp"
      provider = "claude-agent-acp",
    },

    keys = {
      {
        "<C-\\>",
        function() require("agentic").toggle() end,
        mode = { "n", "v", "i" },
        desc = "Toggle Agentic Chat",
      },
      {
        "<C-'>",
        function()
          require("agentic").add_selection_or_file_to_context()
        end,
        mode = { "n", "v" },
        desc = "Add file or selection to context",
      },
      {
        "<C-,>",
        function() require("agentic").new_session() end,
        mode = { "n", "v", "i" },
        desc = "New Agentic Session",
      },
      {
        "<A-i>r",
        function() require("agentic").restore_session() end,
        desc = "Agentic Restore session",
        mode = { "n", "v", "i" },
      },
      {
        "<leader>ad",
        function()
          require("agentic").add_current_line_diagnostics()
        end,
        desc = "Add current line diagnostic to Agentic",
        mode = { "n" },
      },
      {
        "<leader>aD",
        function()
          require("agentic").add_buffer_diagnostics()
        end,
        desc = "Add all buffer diagnostics to Agentic",
        mode = { "n" },
      },
    },
  }

CONFIGURATION                                    *agentic-configuration*

The full default configuration with all options is defined in
`lua/agentic/config_default.lua`. You only need to specify the options
you want to override.

                                           *agentic-config-provider*
provider ~
  Type: `string`
  Default: `"claude-agent-acp"`

  The ACP provider to use. Built-in providers:
  `"claude-agent-acp"`, `"claude-acp"`, `"gemini-acp"`, `"codex-acp"`,
  `"opencode-acp"`, `"cursor-acp"`, `"copilot-acp"`, `"auggie-acp"`,
  `"mistral-vibe-acp"`, `"cline-acp"`, `"goose-acp"`.

                                      *agentic-config-acp-providers*
acp_providers ~
  Type: `table<string, table>`
  Default: see `config_default.lua`

  Customize built-in providers or add new ACP-compatible providers.
  Each entry supports: `name`, `command`, `args`, `env`,
  `default_mode`, `initial_model`.

  `default_mode`   Start every session in this mode (e.g.,
                   `"bypassPermissions"`). Use <S-Tab> in the
                   chat widget to see modes your provider offers.
  `initial_model`  Override the provider's default model (e.g.,
                   `"haiku"`). Use <localLeader>m in the chat
                   widget to see available models.

>lua
  --- @type agentic.PartialUserConfig
  opts = {
    acp_providers = {
      ["claude-agent-acp"] = {
        default_mode = "bypassPermissions",
        initial_model = "haiku",
        env = {
          ANTHROPIC_API_KEY = os.getenv("ANTHROPIC_API_KEY"),
        },
      },
      ["my-custom-provider"] = {
        name = "My Provider",
        command = "/path/to/my-acp-provider",
        args = { "--acp" },
        env = {},
      },
    },
  }
<

                                          *agentic-config-windows*
windows ~
  Type: `table`

  Configure window layout and dimensions.

  `position`           Position of the sidebar: `"right"`, `"left"`,
                       or `"bottom"`. Default: `"right"`.
  `width`              Width of the sidebar. String percentage or
                       number. Default: `"40%"`.
  `height`             Height when position is `"bottom"`. String
                       percentage or number. Default: `"30%"`.
  `stack_width_ratio`  Ratio for stacked panels. Default: `0.4`.

  Sub-panel options (each accepts `win_opts` table):

  `chat.win_opts`          Window options for the chat panel.
  `input.height`           Prompt input height. Default: `10`.
  `code.max_height`        Code block max height. Default: `15`.
  `files.max_height`       File list max height. Default: `10`.
  `diagnostics.max_height` Diagnostics max height. Default: `10`.
  `todos.display`          Show todos panel. Default: `true`.
  `todos.max_height`       Todos max height. Default: `10`.

                                          *agentic-config-settings*
settings ~
  Type: `table`

  `move_cursor_to_chat_on_submit`  Automatically move cursor to chat
                       window after submitting a prompt.
                       Default: `true`.

                                          *agentic-config-hooks*
hooks ~
  Type: `table`

  Callbacks for chat lifecycle events. See |agentic-hooks|.

                                          *agentic-config-headers*
headers ~
  Type: `table`

  Customize window headers for each panel. Each header can be a
  table with `title` and `suffix` fields, or a function that receives
  header parts and returns a custom string.

>lua
  --- @type agentic.PartialUserConfig
  opts = {
    headers = {
      chat = {
        title = "My Chat",
        suffix = "",
      },
      input = function(parts)
        return parts.title .. " | custom"
      end,
    },
  }

USAGE                                                    *agentic-usage*

All functions are accessed via `require("agentic")`.

                                                      *agentic.toggle()*
agentic.toggle({opts})
  Toggle the chat sidebar for the current tabpage.

  Parameters: ~
    {opts}  (table|nil)  Optional fields:
      `auto_add_to_context`  (boolean)  Add selection/file on open.
                                        Default: `true`.
      `focus_prompt`         (boolean)  Move cursor to prompt.
                                        Default: `true`.

                                                        *agentic.open()*
agentic.open({opts})
  Open the chat sidebar. Keeps it open if already visible.
  Accepts the same {opts} as |agentic.toggle()|.

                                                       *agentic.close()*
agentic.close()
  Close the chat sidebar for the current tabpage.

                                               *agentic.add_selection()*
agentic.add_selection({opts})
  Add the current visual selection to the chat context and open
  the sidebar.

  Parameters: ~
    {opts}  (table|nil)  Optional. `focus_prompt` (boolean).

                                                   *agentic.add_file()*
agentic.add_file({opts})
  Add the current file to the chat context and open the sidebar.

  Parameters: ~
    {opts}  (table|nil)  Optional. `focus_prompt` (boolean).

                                *agentic.add_selection_or_file_to_context()*
agentic.add_selection_or_file_to_context({opts})
  Add the visual selection if active, otherwise the current file.

  Parameters: ~
    {opts}  (table|nil)  Optional. `focus_prompt` (boolean).

                                        *agentic.add_files_to_context()*
agentic.add_files_to_context({opts})
  Add a list of file paths or buffer numbers to context.

  Parameters: ~
    {opts}  (table)  Required fields:
      `files`         (table)    List of file paths (strings) or
                                 buffer numbers (integers).
      `focus_prompt`  (boolean)  Optional. Default: `true`.

>lua
  require("agentic").add_files_to_context({
    files = { "src/main.lua", "src/utils.lua" },
  })

  -- By buffer number
  require("agentic").add_files_to_context({
    files = { 1, 5 },
    focus_prompt = false,
  })
<

                                  *agentic.add_current_line_diagnostics()*
agentic.add_current_line_diagnostics({opts})
  Add diagnostics at the current cursor line to context.

  Parameters: ~
    {opts}  (table|nil)  Optional. `focus_prompt` (boolean).

                                      *agentic.add_buffer_diagnostics()*
agentic.add_buffer_diagnostics({opts})
  Add all diagnostics from the current buffer to context.

  Parameters: ~
    {opts}  (table|nil)  Optional. `focus_prompt` (boolean).

                                                 *agentic.new_session()*
agentic.new_session({opts})
  Destroy the current session and start a new one.

  Parameters: ~
    {opts}  (table|nil)  Optional fields:
      `provider`              (string)   Switch provider for the
                                         new session.
      `auto_add_to_context`   (boolean)  Default: `true`.

                                     *agentic.new_session_with_provider()*
agentic.new_session_with_provider({opts})
  Show a provider picker, then create a new session with the
  selected provider.

                                             *agentic.switch_provider()*
agentic.switch_provider({opts})
  Switch to a different ACP provider mid-session. Chat history
  and context are preserved.

  Parameters: ~
    {opts}  (table|nil)  Optional fields:
      `provider`  (string)  Provider name. If omitted, shows a
                            picker.

                                             *agentic.stop_generation()*
agentic.stop_generation()
  Stop the agent's current generation or tool execution. The
  session remains active and ready for the next prompt.

                                             *agentic.restore_session()*
agentic.restore_session()
  Show the provider's session picker to restore a previous
  session. Sessions are interchangeable between Neovim and the
  terminal.

                                              *agentic.rotate_layout()*
agentic.rotate_layout({layouts})
  Rotate through window positions (right -> bottom -> left).

  Parameters: ~
    {layouts}  (table|nil)  List of position strings. Default:
                            `{ "right", "bottom", "left" }`.

KEYMAPS                                                *agentic-keymaps*

These keybindings are automatically set in Agentic buffers:

  Key                    Mode    Description ~
  <S-Tab>                n/v/i   Switch agent mode (if supported)
  <CR>                   n       Submit prompt
  <C-s>                  n/v/i   Submit prompt
  <Tab>                  i       Accept file completion
  <localLeader>p         n       Paste image from clipboard
  <C-v>                  i       Paste image from clipboard
  <localLeader>s         n       Switch ACP provider (keeps history)
  <localLeader>m         n       Switch model (keeps history)
  q                      n       Close chat widget
  d                      n/v     Remove file/selection/diagnostic
  ]c                     n       Next diff hunk
  [c                     n       Previous diff hunk

                                        *agentic-keymaps-customizing*
Customizing keymaps ~

Override default keybindings via the `keymaps` config option:
>lua
  --- @type agentic.PartialUserConfig
  opts = {
    keymaps = {
      widget = {
        close = "q",
        change_mode = {
          { "<S-Tab>", mode = { "i", "n", "v" } },
        },
        switch_provider = "<localLeader>s",
        switch_model = "<localLeader>m",
      },
      prompt = {
        submit = {
          "<CR>",
          { "<C-s>", mode = { "n", "v", "i" } },
        },
        paste_image = {
          { "<localLeader>p", mode = { "n" } },
          { "<C-v>", mode = { "i" } },
        },
        accept_completion = {
          { "<Tab>", mode = { "i" } },
        },
      },
      diff_preview = {
        next_hunk = "]c",
        prev_hunk = "[c",
      },
    },
  }
<

Keymap configuration formats:
  - String: `close = "q"` (normal mode by default)
  - Array: `submit = { "<CR>", "<C-s>" }` (multiple bindings)
  - Table with mode: `{ "<C-s>", mode = { "i", "v" } }`

DIFF PREVIEW                                      *agentic-diff-preview*

When the agent makes file edits, agentic.nvim shows a preview of the
changes before you accept or reject them.

                                        *agentic-config-diff-preview*
Configuration: ~
>lua
  --- @type agentic.PartialUserConfig
  opts = {
    diff_preview = {
      enabled = true,
      layout = "split",  -- "split" or "inline"
      center_on_navigate_hunks = true,
    },
  }
<

  `enabled`                    Enable diff preview. Default: `true`.
  `layout`                     `"split"` (side-by-side) or `"inline"`
                               (unified). Default: `"split"`.
  `center_on_navigate_hunks`   Center view on hunk navigation.
                               Default: `true`.

Navigation: use `]c` / `[c` to move between hunks (configurable
via |agentic-keymaps-customizing|).

Note: changing layout requires restarting Neovim.

                                       *agentic-permission-approval*
Permission approval ~

When a tool call requires permission, the chat buffer shows approval
buttons. Press the corresponding number key for quick responses:
  1 = allow once, 2 = allow always, 3 = reject once, 4 = reject always.

FEATURES                                              *agentic-features*

Slash commands ~
                                                *agentic-slash-commands*
Type `/` in the prompt buffer to see available slash commands with
auto-completion. The `/new` command is always available. Other
commands are provided by your ACP provider.

File picker ~
                                                  *agentic-file-picker*
Type `@` in the prompt to trigger native Neovim completion with
workspace files. Reference multiple files: `@file1.lua @file2.lua`.
Uses `rg`, `fd`, `git ls-files`, or Lua globs as fallback.

Image and screenshot support ~
                                                     *agentic-images*
Drag-and-drop images into the prompt buffer or paste from clipboard.
Drag-and-drop works with any terminal that supports it. Clipboard
paste requires |img-clip.nvim|:
>lua
  {
    "carlos-algms/agentic.nvim",
    dependencies = {
      { "hakonharnes/img-clip.nvim", opts = {} },
    },
  }
<
On Linux, img-clip.nvim requires `xclip` (X11) or `wl-clipboard`
(Wayland). On macOS, `pngpaste`. Windows needs no extra deps.

Session restoration ~
                                            *agentic-session-restore*
Sessions are interchangeable between Neovim and the terminal. Call
|agentic.restore_session()| to see previous sessions from your
provider. If the current tab has an active conversation, you will be
prompted to cancel or clear and restore.

System information ~
                                             *agentic-system-context*
The first message of each session automatically includes platform
info, shell/Neovim version, date, git status, and project root.
This reduces time to first response by providing immediate context.

                                              *agentic-copilot*
Copilot ~

To get Copilot suggestions in the prompt buffer:

copilot.vim: ~
>lua
  {
    "github/copilot.vim",
    init = function()
      vim.g.copilot_filetypes = {
        AgenticInput = true,
      }
    end,
  }
<

copilot.lua: ~
>lua
  {
    "zbirenbaum/copilot.lua",
    opts = {
      should_attach = function(bufnr, bufname)
        if vim.bo[bufnr].filetype == "AgenticInput" then
          return true
        end
        local default =
          require("copilot.config.should_attach").default
        return default(bufnr, bufname)
      end,
    },
  }
<
